# PDF阅读技能

这个技能帮助你高效地阅读和分析PDF文件，提取其中的文本、图片、图表等各种信息。

## 核心功能

1. **文本提取**：提取PDF中的所有文字内容
2. **图片分析**：识别并描述PDF中的图片内容
3. **图表理解**：解读表格、图形、流程图等可视化内容
4. **结构化提取**：提取特定字段、数据或信息片段

## 使用方法

### 1. 读取PDF文件

#### 方法A：使用Read工具（推荐，支持视觉分析）

```
Read: file_path="/path/to/document.pdf"
```

对于大型PDF（超过20页），使用pages参数分段读取：
```
Read: file_path="/path/to/document.pdf", pages="1-20"
```

#### 方法B：使用pdftotext命令行工具（适用于密码保护或纯文本提取）

如果PDF有密码保护或只需要纯文本：

```bash
# 提取全部文本
pdftotext -layout document.pdf -

# 提取指定页码范围
pdftotext -f 100 -l 120 -layout document.pdf -

# 提取到文件
pdftotext -layout document.pdf output.txt
```

**技巧**：结合grep搜索快速定位内容
```bash
pdftotext -layout document.pdf - | grep -n "关键词"
```

### 2. 分析内容

读取后，根据用户需求分析PDF内容：

- **全文阅读**：理解文档整体结构和主要内容
- **针对性提取**：根据用户问题定位特定信息
- **视觉元素**：识别图片、图表，并解释其含义

### 3. 输出格式

根据用户需求，以以下格式输出：

- **文本摘要**：提供文档总结
- **结构化数据**：JSON、表格等形式
- **详细分析**：逐页或逐章节分析
- **问题回答**：直接回答用户的具体问题

## 最佳实践

### 处理大型PDF

如果PDF内容很长（如数据手册、参考手册）：

1. **先读取目录和概述页**（通常是前10-20页）了解文档结构
2. **使用关键词搜索**定位相关章节：
   ```bash
   pdftotext -layout document.pdf - | grep -n "章节标题\|关键词"
   ```
3. **分段读取**：每次读取10-20页，聚焦用户关心的内容
4. **记录页码**：为用户后续查询提供准确的页码参考

### 处理密码保护的PDF

如果遇到密码保护：
1. 首先尝试使用 `pdftotext` 工具，它通常可以绕过只读密码
2. 如果仍然无法读取，告知用户需要未加密版本
3. 用户可以提供其他格式（Word、HTML、文本）作为替代

### 阅读技术文档（数据手册、参考手册）

对于芯片手册、技术规格书等技术文档：

1. **识别文档类型**：
   - 数据手册（Datasheet）- 关注电气参数、引脚定义、时序图
   - 参考手册（Reference Manual）- 关注寄存器描述、工作原理
   - 用户手册（User Manual）- 关注使用方法、配置步骤

2. **提取关键信息**：
   | 信息类型 | 提取内容 |
   |----------|----------|
   | 寄存器 | 名称、地址、位定义、默认值 |
   | 电气参数 | 电压范围、电流、频率、温度 |
   | 引脚 | 引脚号、功能、电气特性 |
   | 时序 | 时钟周期、建立时间、保持时间 |

3. **以结构化表格呈现**：技术数据用表格比文字更清晰

### 提取图片和图表

PDF中的图片和图表会以视觉形式呈现：

1. **图片**：描述图片内容、标注的文字说明
2. **表格**：提取表头、行列数据，以Markdown表格呈现
3. **图表**：
   - 图表类型（柱状图、饼图、折线图等）
   - 坐标轴含义和单位
   - 数据点和趋势
   - 图例说明
4. **流程图**：描述流程步骤、判断条件、数据流向
5. **电路图/引脚图**：描述引脚编号、功能分配、连接关系

### 结构化提取

当用户需要特定信息时：

1. **先了解PDF的整体结构** - 目录、章节划分
2. **定位相关信息** - 使用搜索或逐页扫描
3. **提取并以结构化格式返回**：
   - **配置参数表**：参数名、取值范围、默认值、说明
   - **寄存器表**：地址、名称、位域、读写属性
   - **引脚表**：引脚号、名称、功能、电气特性
   - **命令/指令表**：命令码、参数、返回值、功能

### 输出质量检查清单

输出前确认：
- [ ] 是否提取了用户要求的所有信息？
- [ ] 数据是否以清晰的表格/列表呈现？
- [ ] 页码引用是否准确？
- [ ] 技术术语是否准确？
- [ ] 单位和数值是否正确？

### 场景4：阅读技术数据手册（实际案例）
```
用户：阅读CH585数据手册的ADC相关内容

操作步骤：
1. 由于PDF有160页且有密码保护，先尝试pdftotext工具：
   pdftotext -layout CH585DS1.PDF -

   pdftotext -layout CH585DS1.PDF - | grep -n "ADC"

3. 定位到第105-113页为ADC章节，分段提取：
   pdftotext -f 105 -l 113 -layout CH585DS1.PDF -

4. 提取关键信息并整理：
   - 12位分辨率、17个通道
   - 寄存器列表（地址、功能）

   - 采样时间、时钟配置
   - 工作模式（单次/连续/DMA/扫描）

```

## 注意事项

### 常见问题及解决方案

| 问题 | 解决方案 |
|------|----------|
| **PDF有密码保护** | 使用 `pdftotext` 命令行工具，它通常可以绕过只读密码 |
| **PDF页数过多** | 使用 `pages` 参数分段读取，或使用 `pdftotext -f -l` 指定页码范围 |
| **中文乱码** | PDF中的文本可能已经编码，尝试不同编码或直接使用pdftotext |
| **扫描件** | 如果是纯图片扫描件，文字无法直接提取，需要告知用户使用OCR工具 |
| **复杂布局** | 某些PDF布局复杂（如多栏），可能需要分段阅读 |
| **图片质量** | PDF中的图片质量取决于原文件，分析时请注意清晰度 |

### 工具选择指南

| 场景 | 推荐工具 | 原因 |
|------|----------|------|
| 需要分析图片/图表 | Read工具 | 支持视觉分析 |
| 只需要文本内容 | pdftotext | 快速、完整 |
| PDF有密码保护 | pdftotext | 可绕过只读密码 |
| 大型PDF（>100页） | pdftotext + grep | 快速定位内容 |
| 需要精确页码 | pdftotext -f -l | 精确控制范围 |
